Overview

An asset is a video file that can be contained within a Franchise, Show, Season, Special, and Episode. Assets can be full length episodes, clips, or previews. Because video clips and previews (not full length episodes) can be considered "Extras" in their own section on PBS.org, Media Manager provides the ability to create Assets and Extras in two separate places in the console.

Episodes and Specials can contain only one full length episode but may include unlimited clips and previews. Once you have added a full length episode, you cannot add another to the same object. 

Media Manager optimizes the raw video file that you add to your asset and automatically generates several formats to adapt to a user's internet speed. So if a user with a slow internet connection clicks on your video, they are automatically served the file size that accommodates their internet connection speed.

How to add notes

The Notes feature provides a place for editors to store internal notes and collaborate with colleagues. Notes is an optional feature intended as a convenience for console users and is not included in the API or displayed on frontend platforms. The text limit is 2,000 characters, including spaces, and can contain text only (hyperlinks can be added but they will not be clickable).

To use the Notes feature, simply click Notes at the top of the page and begin typing. The Notes bar displays in yellow when content has been added. If no notes are present, the bar displays in grey. When finished, click Save Notes to save your changes or Delete Notes to clear the entire field. The Notes feature contains its own save buttons that are separate of the objects save buttons. So if you save the page but you dont save the Notes section, your notes are not saved.

Notes are read-only for the viewer role.

Alternate Accessibility Versions

In the Media Manager console, you can relate multiple, separate instances of the same video content, each with different enhanced accessibility features and different source video files. This helps users on pbs.org easily navigate between the related versions within the video player. (A icon displays in the player control area, when one or more enhanced accessibility versions are available for a video.) 

What is Enhanced Accessibility?
An accessibility option which is unique to a particular video, and/or may result in a unique video duration. The Enhanced Accessibility options in Media Manager include:

  • American Sign Language (ASL): A visual language, where information is conveyed using shape, placement, and movement of the hands, as well as facial expressions and body movements.

  • Extended Audio Description (EAD): With standard Audio Description (AD), descriptions fit within the existing, natural pauses in the video. With Extended Audio Description (EAD), longer pauses are added within the video, to provide opportunities for lengthier descriptions and more context. EAD videos are typically longer in duration.

  • Open Captions (OC): Captions which are burned into the video file and are always displayed during playback. Open Captions cannot be customized or turned off by the user.

    • NOTE: To comply with FCC caption requirements, a video with Open Captions can be provided as an alternate version, as long as there is another version available which has standard, customizable, and properly position captions. 

Why do videos with Enhanced Accessibility need to be separate versions (i.e. separate Media Manager video assets)?
Each version typically has a unique source video file, which can vary based on the content and duration. A Media Manager video asset can only accept one video file.

Standard accessibility options can be managed within a single Media Manager asset because there is a single source video file, with a single duration. Standard options include: Multiple caption files for different languages, and multiple audio tracks for AD and different languages .

Example use case for relating enhanced accessibility videos

Say you have four (4) separate Media Manager video assets which all represent the same content, but the video duration and enhanced accessibility feature for each one is different:

  1. One instance has no enhanced accessibility options and the duration is 45 min. (It would have at least some of the standard accessibility options/requirements, such as one or more caption files, and one or more audio tracks included in the video file.)

  2. One instance has EAD (Extended Audio Description) and the duration is 60 min. (It's longer because extra pauses are added to provide more opportunities for audio description.)

  3. One instance has ASL (American Sign Language) and the duration is 45 min. (Its duration is the same as #1 but the video is presented with sign language, so it's a different video file.)

  4. One instance has OC (Open Captions) and its duration the 45 min. (The duration is the same as #1 but the video has burned-in captions which cannot be removed or customized. Open Captions may be preferred for some users, but they do not satisfy the FCC's caption requirements for standard, customizable, and properly positioned captions. The Open Caption video instance could only be provided as an alternate version.

How to relate videos with Enhanced Accessibility features

  1. Go to the Related Content tab for one of the videos. This is the (only) video where you will manage the relationships. For the example above, it's recommended to start in the non-enhanced version (#1), but it's not a requirement.

  2. Scroll down to the Alternate Accessibility Versions section.

  3. Add one related video at a time, entering its asset CID, and selecting the accessibility type.

    • The accessibility type for the initial video must also be selected.

How to tell if a video on pbs.org has enhanced accessibility versions

On pbs.org, when one or more enhanced accessibility versions are available for a video, a icon displays in the player control area.

  • All Passport members

  • Your Station's Passport members

Caption offset correction feature

This feature targets a particular scenario which results in a failed caption ingestion. This pertains to uploaded SCC caption files which are invalid due to a start time that is advanced by one hour. (Meaning the captioning and video/audio timing would be mis-aligned by one hour.)

This timing is common (and valid) for SCC caption files used for broadcast, but it needs to be corrected (time shifted back by one hour) before getting uploaded in MM, for streaming.

How it works: If an SCC caption file is uploaded in the console which has the (invalid) one-hour advanced start time, an error message will display and present the user with an option to allow MM to programmatically correct the caption timing (shift it back one hour), and re-ingest the file. (Alternatively, the user can choose to correct the file themselves and re-upload it at a later time.)

If the user elects to proceed with the process, the caption file archived on S3 will be the corrected version. This is done so that if the user chooses to reingest the file again at a later time, the corrected one will be used without error.

This feature is available in the MM console, but not in the API.

Important: If the user chooses to proceed with the automatic offset correction feature, its imperative that they subsequently verify the audio and closed captioning are accurately synced throughout the video, after ingestion is complete. Media Manager cannot verify synchronization, so this step must be completed to discharge our legal due diligence responsibility.

Learn more about auto-publish

What is auto-publish?

The auto-publish feature ensures that once all necessary files have been successfully ingested, your asset will automatically publish. This means that you can upload your video file (and caption file if applicable) and walk away during the ingestion process to do other things. You must check back in on the process in case any errors occur during ingestion. If errors occur, the process will stop and your asset will not publish until the errors are corrected.

What if I leave and my asset encounters an error?

It's very important to remember that when you are using the auto-publish feature, you must be diligent about checking the ingestion status.  If ingestion fails, an error message appears in the console and your asset does not publish. You need to manually fix the errors and republish the asset by clicking the Publish button.  Refer to the publish status to see if everything successfully ingested and published.

Support for Audio Description and Secondary Languages

Media Manager supports video source files which include one or more audio tracks. (See Full Specs.)

For example, the video source file may have English as the primary audio track, and there could be additional tracks included for English Audio Description (AD), and other languages such as Spanish, French, etc.

What is Audio Description (AD): Additional audio content, inserted as a narrative between the dialogue and other program sounds, which describes visual aspects of the video that may not be accessible to some visually-impaired users. Audio Description is provided as a separate track, and users can choose to turn it on in addition to the primary audio track.

At this time, not all PBS platform players can support audio track selection, but this will be improved over time. See Full Specs to get more information about player support.)

Detected audio details are displayed for videos ingested on/after April 12, 2021 (when the support for multiple audio tracks was introduced). This can be found in the expandable Details area for the video file on an asset's Video tab.

  • Note: For videos ingested prior to the feature release, the audio details will not be displayed in the console. In the future, we will determine how to handle populating the audio details for pre-existing video files. Stay tuned.

At this time, Media Manager only supports uploading a single closed captioning file, regardless of the number of audio tracks.

Get the Full Specs and learn more about uploading video files containing multiple audio tracks.

How to reingest a video file 

In some cases where, for example, an older video is lacking some of the modern output encodings, or if an output encoding fails to get generated during the initial transcode process, you can reingest the existing video file. Only video files that have an associated archive file available at its S3 location can be reingested. If the archived video file is glacierized in S3, the reingestion process takes about 5-12 hours to complete. The process is automated and video playback is not disrupted during the reingestion process.

Reingesting a video file is not intended for general troubleshooting or for bulk tasks. Please check the following before reingesting a file:

  • View the video on multiple devices and browsers

  • Check your internet connection to see if you have a slow connection

  • Verify that the video is published

  • Verify that the caption files are correctly ingested

  • Verify that the output files have been generated 

The steps below illustrate how to reingest a video file in Media Manager.

What does glacierize mean?

When a video is glacierized, it goes into an Amazon Glacier storage service that is optimized for infrequently used data, or "cold data." This allows us to store archived video files at a reduced cost, and only request access to them as needed. Getting access to glacierized files can take up to 12 hours. All video archive files get stored in Amazon Glacier shortly after being ingested in Media Manager.
